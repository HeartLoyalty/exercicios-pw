Guia de como usar o Terminal Bash da forma mais basica

    Clone

  • Pegar o Url do seu repositório para baixar ele dentro da sua maquina.

    • Usando bash pela primeira vez na sua maquina

  • git config Os comandos git config são usados para configurar a identidade de quem está fazendo o commit. Todo commit é assinado com um nome e um e-mail. Você precisa configurar sua identidade globalmente. Use (--global) para que esta identidade seja usada em todos os seus repositórios naquela máquina. Define o nome que aparecerá nos seus commits (git config --global user.name "Seu Nome Completo") efine o e-mail que aparecerá nos seus commits (deve ser o e-mail da sua conta GitHub) (git config --global user.email "seu.email@exemplo.com")

    • Altere o Nome do Branch Principal (Opcional, mas Recomendado)

  • Se a sua branch principal local ainda for chamada master, é uma boa prática renomeá-la para main (o nome padrão moderno usado pelo GitHub). Renomeia a branch local atual para 'main' git branch -M main

    • Notas

  • As informações contidas aqui é um método que eu aprendi e que na minha visão é a forma mais simples de se subir arquivos no github usando o bash. Existem muitas outras formas de se fazer umas sem usar o terminal, outras usando completamente o terminal desde a criação dos diretorios até, criação de pastas, de arquivos até vc subir para o GitHub. Me esforcei ao maximo para deixar esse guia de um jeito simples de entender, eu mesmo ainda sei muito pouco sobre tudo o que o Git pode fazer, mas esse pouco que sei estou compartilhando para que cada vez mais pessoas possam estar conhecendo e aprendendo. para que nos tornemos desenvolvedores mais completos num futuro proximo.

Criando um Clone do Seu Repositório

  1. Após pegar o link, entre no Seu VSCODE, e sem entrar em nenhuma pasta ou arquivo, na tela inicial. use o comando Crtl+Jpara abrir o terminal da plataforma.
  2. Com o terminal aberto procure pelos icones (+) com uma seta para baixo e nele selecione o terminal bash
  3. No bash escreva o comando Git Clone + Url do repositório que vc copiou anteriormente

Alocando Seu Repositório Clonado no Diretorio Desejado

  1. A partir deste ponto o Repositório que você Criou no Git Hub está baixado na sua Maquina
  2. Tenha em mente que todo o terminal vai abrir a pasta local do seu Usuario na maquina que vc estiver usando. caso queira salvar o Clone em um PenDrive ou em uma pasta fora do seu Usuario você deve navegar até ela escrevendo o comando (cd) no terminal e o local onde vc quer clonar o arquivo.

(cd ..) para voltar um Diretório do que está no momento

(cd/caminho/para/o/diretorio/)

Exemple: (cd C:/Users/seu_usuario/Downloads)

Comandos para atualizar seus Projetos pelo terminal

Nesse ponto você já seguiu todos os passos para criar seus projetos e agora vc quer adicionar eles no seu repositório lá no Seu GitHub

  1. Escreva no Terminal: Git Init quando esse comando for acionado ele ele cria uma subpasta oculta chamada .git no diretório atual. Após o git init, o Git está pronto para rastrear arquivos, mas nenhum arquivo é rastreado automaticamente
  2. Após Escreva no Terminal: Git pull este comando Sincroniza seu repositório local com as últimas alterações do repositório remoto (como o GitHub). Antes de começar a trabalhar e, especialmente, antes de enviar suas próprias alterações, para garantir que você esteja trabalhando com a versão mais recente do projeto.

    3. Feito isso, você já pode fazer todas as modificações necessarias no seu projeto. adicionar arquivos, pastas e etc

  3. Agora Vamos adicionar as modificações que voce fez usanod git add . com esse comando todas as alteraçõesque vc fez serão adicionadas para serem commitadas

Subindo arquivos pro GitHub

  1. Arquivos adicionados hora de usar o Comando git commit -m'descriçao do commit' usando isso Cria uma "fotografia" (um commit) das mudanças que estão na Staging Area e as armazena permanentemente no histórico do seu repositório local. É obrigatório incluir uma mensagem (-m) que descreva o que foi alterado.
  2. Após o commit para finalizarmos o processo use o comando git Push Envia seus commits locais (que você acabou de criar) para o repositório remoto (o GitHub).
